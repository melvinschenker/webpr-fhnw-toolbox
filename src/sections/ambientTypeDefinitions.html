Zentrales Typing

Das Prinzip nennt sich Ambient Type Definitions. Du lagerst alle Datentypen in eine separate Datei aus, damit dein eigentlicher Code sauber bleibt.

  1. Erstellen: Lege eine Datei types.js (oder types.d.ts für bessere IDE-Erkennung) im Projektordner an.
  2. Definieren: Schreibe dort nur JSDoc-Kommentare (@typedef) rein. Kein ausführbarer Code.
  3. Nutzen: In anderen JS-Dateien kannst du die Typen via JSDoc referenzieren (moderne IDEs wie VSCode unterstützen das automatisch).

types.js

/**
	* @typedef {Object} Product
	* @property {number} id - Eindeutige Produkt-ID
	* @property {string} name - Name des Produkts
	* @property {number} price - Preis in CHF
*/

main.js

/**
	* Berechnet den Gesamtpreis aller Produkte im Warenkorb.
	* @param {Product[]} cart - Array von Produkten im Warenkorb
	* @returns {number} Gesamtpreis
*/
function calculateTotal(cart) {
	// Automatische Typprüfung für Array-Elemente ('cart' ist ein Array von 'Product')
	return cart.reduce((total, item) => total + item.price, 0);
}

Warum das funktioniert:Die IDE indiziert das gesamte Projektverzeichnis für IntelliSense. Dem Browser ist es egal, da es sich nur um Kommentare handelt - zur Laufzeit existieren diese Typen nicht.
