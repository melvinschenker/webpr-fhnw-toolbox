Async/Await Essentials

Async/Await ist moderner JavaScript für asynchrone Operationen. Mit await wird auf Promises gewartet, ohne komplexe .then()-Ketten schreiben zu müssen. Die Wahl zwischen sequenzieller und paralleler Ausführung hat großen Einfluss auf Performance.

Sequenzielle vs. Parallele Ausführung

// SEQUENZIELL (hintereinander): ~3 Sekunden
// Jeder Fetch wartet, bis der vorherige abgeschlossen ist
async function sequentially() {
    const data1 = await fetch('https://api.example.com/data/1').then(r => r.json());
    const data2 = await fetch('https://api.example.com/data/2').then(r => r.json());
    const data3 = await fetch('https://api.example.com/data/3').then(r => r.json());
    return [data1, data2, data3];
}

// PARALLEL (gleichzeitig): ~1 Sekunde
// Alle Fetches starten gleichzeitig, Promise.all() wartet auf alle
async function inParallel() {
    const [data1, data2, data3] = await Promise.all([
        fetch('https://api.example.com/data/1').then(r => r.json()),
        fetch('https://api.example.com/data/2').then(r => r.json()),
        fetch('https://api.example.com/data/3').then(r => r.json())
    ]);
    return [data1, data2, data3];
}

Array mit map() durchlaufen

const ids = [1, 2, 3];

// Mit map() und Promise.all() (PARALLEL)
const dataParallel = await Promise.all(
    ids.map(async id => {
        const response = await fetch(`https://api.example.com/data/${id}`);
        return response.json();
    })
);

// Mit for...of Loop (SEQUENZIELL)
const dataSequential = [];
for (const id of ids) {
    const response = await fetch(`https://api.example.com/data/${id}`);
    dataSequential.push(await response.json());
}
