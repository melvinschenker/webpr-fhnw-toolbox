eval() vs Function() Konstruktor

Ein kritischer Unterschied bei der Ausführung von dynamischem Code: eval() läuft im lokalen Scope und kann Seiteneffekte verursachen, Function() läuft im globalen Scope und ist sicherer.

Beispiel: Sichere Formel-Auswertung

❌ Problem mit eval()

let adminAccess = false;
const formula = "adminAccess = true; x * 2";
eval(formula); // adminAccess ist jetzt true!

✅ Lösung mit Function()

let adminAccess = false;
const func = new Function('x', 'adminAccess = true; return x * 2');
func(5); // 10 adminAccess bleibt false!

Merksätze

  • eval() = gefährlich, weil es alles im lokalen Kontext kann
  • Function() = sicher, weil es nur mit Parametern arbeitet
  • Für User-Input: immer Function() verwenden!
